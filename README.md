# Data Visualization and Modeling - Campus

This repository contains the campus version of the Data Visualization and
Modeling course notebooks.

Campus notebooks include blank activity cells for in-class practice. Your
instructor may complete these cells during class or ask you to complete them on
your own.

## What Is In This Repository

- Weekly lecture notebooks
- Weekly lab notebooks
- Supplemental notebooks
- Datasets and support files used by the notebooks

Generated Python `.py` source files and private instructor notes are not
included in this student-facing repository.

## Recommended Setup

Install these before working with the notebooks:

- [Visual Studio Code](https://code.visualstudio.com/)
- [Git](https://git-scm.com/downloads)
- [Anaconda](https://www.anaconda.com/download)
- VS Code extensions:
  - [GitHub Pull Requests and Issues](https://marketplace.visualstudio.com/items?itemName=GitHub.vscode-pull-request-github)

Use Anaconda Navigator and JupyterLab to open and run notebooks. Use VS Code to
clone the repository, commit your work, and push changes to GitHub.

## Fork This Repository

Forking creates your own copy of this repository in your GitHub account.

1. Sign in to GitHub.
2. Open this repository on GitHub.
3. Select **Fork** in the upper-right corner.
4. Choose your GitHub account as the owner.
5. Select **Create fork**.

Work from your fork so you can save your own notebook changes.

If your instructor gives you a term branch name such as `term/2026-07`, use
that branch for the current campus section.

## Clone Your Fork In VS Code

1. Open VS Code.
2. Open the Command Palette:
   - Windows: `Ctrl+Shift+P`
   - macOS: `Cmd+Shift+P`
3. Run `Git: Clone`.
4. Choose **Clone from GitHub** if prompted.
5. Sign in to GitHub if VS Code asks.
6. Select your fork of this repository.
7. Choose a folder on your computer where VS Code should save the files.
8. Select **Open** when VS Code asks whether to open the cloned repository.

You should now see the course folders in the VS Code Explorer. VS Code is the
tool you will use for GitHub syncing.

## Open The Notebooks In JupyterLab

1. Open **Anaconda Navigator**.
2. Launch **JupyterLab**.
3. In JupyterLab, browse to the folder where you cloned your GitHub fork.
4. Open the course folder.
5. Open the `.ipynb` notebook for the lesson or lab you are working on.

Run notebook cells from top to bottom unless your instructor says otherwise.

## Save Your Work To GitHub

After you edit notebooks in JupyterLab:

1. Open the **Source Control** view in VS Code.
2. Review your changed files.
3. Enter a short commit message.
4. Select **Commit**.
5. Select **Sync Changes** or **Push** to send your work to your GitHub fork.

Commit and push regularly so your notebook work is backed up online.

## Keep Your Fork Updated

If your instructor updates the course repository, GitHub may show a **Sync
fork** button on your fork. Use it to bring your fork up to date before starting
new work.

If you are unsure whether to sync, ask your instructor first, especially if you
have uncommitted notebook work.

## Working With Notebooks

- Open and run `.ipynb` files in JupyterLab from Anaconda Navigator.
- Use VS Code for GitHub syncing, not for running notebook lessons.
- Keep datasets in their existing folders so relative paths continue to work.
- Do not rename course folders unless your instructor asks you to.

## Need Help?

If cloning, signing in, pushing to GitHub, or running notebooks does not work,
ask your instructor and include:

- A screenshot of the error
- The notebook or file you were working in
- Whether you are using Windows or macOS
